# Sales One-Pager — рынок РФ

**Продукт:** Grounded LLM — HR Policy Assistant  
**Формат:** on-prem / private cloud, пилот 8 недель → годовая лицензия  
**Аудитория:** HR-директор, руководитель IT, ИБ на первом созвоне

> **One-liner:**  
> *Внутренний HR-ассистент по вашим регламентам — ответы только из документов, с цитатами, в вашей инфраструктуре. Не ChatGPT.*

---

## 1. Кому продаём (ICP)

| Параметр | Ориентир |
|----------|----------|
| Размер | 200–5 000 сотрудников |
| Покупатель | HR Director / Head of People + IT sponsor |
| Боль | Одни и те же вопросы в HR; публичный ChatGPT запрещён политикой ИБ |
| Канал для сотрудников | Web-чат и/или **Telegram** (Mini App) |
| Инфра | Docker на VM клиента или private cloud (Yandex Cloud, Selectel, on-prem) |

**Хороший fit:** mid-market, IT может выделить VM, есть HR handbook в PDF/DOCX.  
**Сложный fit:** банки/гос без SSO и audit log — см. раздел «Честные ограничения».

## 2. Оффер

| Пункт | Детали |
|-------|--------|
| Область | 1 домен знаний (HR / employee policies) |
| Документы | До 50 файлов или 200 МБ (`.txt`, `.pdf`, `.docx`) |
| Пользователи | До 500 активных на пилоте |

**Demo-ready:** конец 3-й недели пилота (после загрузки документов клиента и reindex).

## 4. Pitch для HR (2–3 минуты)

**Проблема.**  
Сотрудники каждую неделю спрашивают про отпуск, больничные, удалёнку. HR тратит часы на повторы. ChatGPT и публичные боты для внутренних политик нельзя — данные и политики не должны уходить наружу.

**Решение.**  
Grounded LLM — ассистент, который отвечает **только из ваших документов**: регламенты, handbook, локальные приказы. К каждому ответу — **цитата с именем файла**. Если ответа нет в базе — честное «нет в документах», без выдумок.

**Почему безопасно для IT.**  
Разворачивается **у вас**: Postgres, индекс, файлы — на ваших серверах. Embeddings считаются локально в контейнере. LLM — на ваш выбор: локальная модель, Yandex GPT, GigaChat или другой OpenAI-compatible endpoint. Мы не обучаем модели на ваших данных.

**Следующий шаг.**  
30-мин demo → пилот 8 недель с измеримыми KPI → годовая лицензия.

## 5. Pitch для IT / ИБ (1 минута)

- **Деплой:** Docker Compose (4 сервиса: postgres, Go API, Python RAG, webapp).
- **Данные:** `data/{tenant}/{domain}/`, Chroma, Postgres — volumes клиента.
- **Наружу** (только если выбран облачный LLM): текст вопроса + фрагменты chunks, не полный корпус.
- **Embeddings:** `multilingual-e5-small` внутри Python-контейнера, без внешнего API.
- **Auth:** Telegram Web App HMAC, API keys для интеграторов, Basic Auth в админке.
- **Multi-tenant:** `X-Tenant-ID`, изоляция в Chroma и сессиях.

Подробнее: [SECURITY_BRIEF.md](./SECURITY_BRIEF.md).

## 6. LLM для РФ — что сказать на созвоне

| Вариант | Когда подходит | Риск для ИБ |
|---------|----------------|-------------|
| **Локальный vLLM / Ollama** на GPU клиента | Air-gap, максимальный контроль | Минимальный — трафик не уходит |
| **Yandex GPT / GigaChat** (OpenAI-compatible) | Данные в РФ, привычный вендор | Договор с провайдером, DPA |
| **OpenRouter / OpenAI** | Только если у клиента разрешён egress | Часто **блокер** в РФ |

В `.env` клиента:

```env
DEFAULT_LOCALE=ru
LLM_BASE_URL=<endpoint клиента>
LLM_MODEL=<модель>
LLM_API_KEY=<ключ>
TELEGRAM_AUTH_DISABLED=false   # в prod — только Telegram или API keys
```

**Фраза для слайда:** «Мы не привязаны к OpenAI — любой OpenAI-compatible endpoint, включая полностью локальный.»

## 7. Чеклист для ИБ (перед пилотом)

Используйте на kickoff (Week 1). Отмечайте вместе с клиентом.

### Данные и 152-ФЗ

- [ ] Где физически размещены VM / volumes (ЦОД клиента, регион)
- [ ] Какие ПДн могут попасть в чат (ФИО в вопросах — редко; основной контент — политики)
- [ ] Нужен ли локальный LLM (без передачи текста за рубеж)
- [ ] Политика хранения истории чата (Postgres) — срок retention (настраивается на Phase B; на пилоте — согласовать вручную)
- [ ] Backup: Postgres + Chroma + `data/` — кто делает, как часто

### Доступ

- [ ] Канал входа: Telegram Mini App / web / API only
- [ ] `TELEGRAM_AUTH_DISABLED=false` в production
- [ ] Сильные `ADMIN_PASSWORD`, `ADMIN_SECRET`
- [ ] Admin UI и `/metrics` — VPN или ACL на reverse proxy
- [ ] TLS на reverse proxy (Let's Encrypt / корпоративный сертификат)
- [ ] `CORS_ALLOWED_ORIGINS` — только домены клиента

### LLM и субпроцессоры

- [ ] Выбран провайдер LLM и подписан DPA (если облако)
- [ ] Зафиксировано: в LLM уходят **фрагменты**, не весь corpus
- [ ] Fine-tuning на документах клиента **не выполняется** (архитектура продукта)

### Эксплуатация

- [ ] Rate limits (`RATE_LIMIT_REQUESTS_PER_MINUTE`)
- [ ] Ротация API keys
- [ ] Секреты не в git
- [ ] Runbook reindex и восстановление после сбоя

### Что попросят позже (Phase B — не обещать на пилоте)

- SSO (OIDC/SAML)
- RBAC (роли editor / admin / chat-only)
- Audit log UI (upload, delete, reindex, login)
- Analytics dashboard

## 8. Demo 30 минут (русская локаль)

Полный сценарий: [domain-packs/DEMO_SCRIPT_RU.md](./domain-packs/DEMO_SCRIPT_RU.md).

### Подготовка (за 15 мин до созвона)

```bash
cp .env.example .env
# В .env:
#   DEFAULT_LOCALE=ru
#   TELEGRAM_AUTH_DISABLED=true   # только для demo в браузере
#   LLM_API_KEY=...
#   LLM_BASE_URL=...

docker compose up -d --build
python scripts/reindex_rag.py

# Проверка качества (опционально, 12 кейсов):
pip install requests
python scripts/run_rag_eval.py --suite default
```

Открыть: `http://localhost/?locale=ru`  
Админка: `http://localhost/admin.html`

### Структура созвона

| Блок | Время | Содержание |
|------|-------|------------|
| Проблема | 3 мин | Повторы в HR, ChatGPT нельзя |
| Live chat | 10 мин | 8 вопросов из demo script, показать Sources |
| Доверие | 5 мин | Цитаты, verify, «нет в базе» без галлюцинаций |
| Админка | 5 мин | Upload → Reindex → новый вопрос |
| Enterprise path | 5 мин | API keys, OpenAPI, multi-tenant, roadmap |
| Закрытие | 2 мин | Пилот 8 нед, KPI, цена в ₽ |

### 8 вопросов для live demo

| # | Вопрос | Ожидаемый факт |
|---|--------|----------------|
| 1 | Сколько дней оплачиваемого отпуска положено сотруднику? | 28 |
| 2 | За сколько дней нужно запланировать отпуск в HR Portal? | 14 |
| 3 | Сколько дней отпуска можно перенести на следующий год? | 14 |
| 4 | До которого часа нужно уведомить руководителя о болезни? | 10:00 |
| 5 | Как оформить больничный лист? | 3 рабочих дня в HR |
| 6 | Можно ли работать удалённо два дня в неделю? | 2 дня |
| 7 | В какие дни рекомендуется быть в офисе? | вторник, четверг |
| 8 | В какие часы сотрудник должен быть доступен в мессенджере? | 09:00–18:00 |

**Вне базы (доверие):**

- «Какой оклад у CEO на Луне в 2099 году?» → нет в базе  
- «Сколько дней отпуска у сотрудника на Марсе?» → нет в базе

## 9. KPI пилота (договорить заранее)

| Метрика | Цель |
|---------|------|
| In-scope ответы **с цитатами** | ≥85% |
| Verify pass (цифры/факты) | ≥75% |
| 👍 на сообщения ассистента | ≥70% |
| P95 время ответа | <15 с |

«Нет в базе знаний» — **корректное** поведение, не провал.

## 10. Честные ограничения (не обещать)

| Не входит в пилот | Когда |
|-------------------|-------|
| SSO, RBAC, audit log UI | Phase B |
| Реестр отечественного ПО «из коробки» | Отдельная упаковка / клиент |
| Готовый пакет 152-ФЗ | Частично закрывается on-prem + локальный LLM; полный compliance — с юристом клиента |
| Расчёт зарплаты, кадровые решения | Вне scope |
| 24/7 SLA, кастомные интеграции (1C, SAP) | Отдельный SOW |
| Написание контента handbook | Клиент или отдельный проект |

## 11. Возражения и ответы

| Возражение | Ответ |
|------------|-------|
| «Это просто ChatGPT?» | Нет. Ответ только из ваших документов + цитата. Без базы — не выдумывает. |
| «Данные утекут в OpenAI» | Деплой у вас. LLM — ваш endpoint, в т.ч. локальный. В промпт — фрагменты, не архив. |
| «Сотрудники не зайдут в ещё один портал» | Telegram Mini App — привычный канал; можно API в intranet. |
| «Нужен SSO» | На пилоте — Telegram/API keys. SSO в roadmap Phase B; можно заложить в контракт post-pilot. |
| «Дорого» | Сравнить с FTE HR на повторяющихся вопросах; пилот фиксирован, KPI измеримы. |
